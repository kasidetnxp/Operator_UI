# ระบบจัดการ FPC - NXP WT (FPC Management System - NXP WT)

ระบบอินเทอร์เฟซผู้ปฏิบัติงาน (Operator User Interface) ที่มีความทันสมัยและตอบสนองการใช้งานบนทุกอุปกรณ์ (Responsive Layout) สำหรับควบคุมใบงานส่งรถนำทางอัตโนมัติ (AGVs) เพื่อทำรายการเบิก, คืน และสลับเปลี่ยน **Front Opening Pod Carriers (FPCs)** ระหว่าง **คลังสมาร์ท (Smart Storage)** และเครื่องจักรผลิต/เวิร์กสเตชันในโรงงาน

---

## 🚀 เทคโนโลยีหลักที่ใช้พัฒนา (Technical Stack)

- **แกนหลัก**: React 18.3.1 (TypeScript)
- **ระบบบิลด์และรัน**: Vite 8.x
- **สไตล์การตกแต่ง**: Tailwind CSS v4 (พร้อมกำหนดคีย์ธีมคัสตอมใน `@theme`) และ Vanilla CSS
- **ส่วนประกอบ UI**: Material-UI (MUI v7) และ Radix UI primitives
- **ไอคอน**: Lucide React
- **กราฟและสถิติ**: Recharts (สำหรับการวิเคราะห์คิวงาน)
- **แอนิเมชัน**: Motion (Framer Motion v12)
- **ป๊อปอัปแจ้งเตือน**: Sonner (ระบบแสดงสถานะแบบ Toasts)

## ✨ ฟีเจอร์หลักในระบบ (Key Features)

1. **การยืนยันตัวตนของพนักงาน (Employee Authentication)**
   - หน้าจอล็อกระบบโดยกำหนดให้กรอกรหัสพนักงานและรหัสผ่านเพื่อยืนยันตัวตน
   - แบ่งสิทธิ์เข้าถึงฟังก์ชันการทำงานตามบทบาท (Admin, Store, Operator)
   - จัดการเซสชันการล็อกอินและลำดับการออกจากระบบที่ปลอดภัย

2. **โหมดการทำงาน 4 รูปแบบ (LOAD / UNLOAD / Swap / UNLOAD & LOAD)**
   - **LOAD (คืน FPC)**: สั่งรถ AGV ไปรับ FPC จากเครื่องจักรและนำกลับคืนไปเก็บที่ Smart Storage
   - **UNLOAD (เบิก FPC)**: ค้นหาและเลือก FPC จากคลัง Smart Storage และสั่งรถ AGV ไปจัดส่งที่เครื่องจักรปลายทาง
   - **สลับ FPC (Swap)**: โอนย้าย FPC ระหว่างเครื่องจักรผลิตด้วยกันโดยตรงโดยไม่ต้องวิ่งกลับคลัง
   - **UNLOAD & LOAD (เปลี่ยน FPC)** (เพิ่มใหม่): ถอด FPC ตัวเดิมออกจากเครื่องจักร และทำการติดตั้ง FPC ตัวใหม่จากคลังทดแทนในทันทีในรอบใบงานเดียวกัน โดยระบบจะอัปเดตและสลับตำแหน่งของ FPC ทั้งสองตัวในฐานข้อมูลจำลองเมื่อจบงาน

3. **ฟังก์ชันเลือกเครื่องจักร (Machine Selector)**
   - ค้นหาและคัดกรองสถานะแบบเรียลไทม์สำหรับเครื่องจักรที่ลงทะเบียนในระบบกว่า 50 เครื่อง (`AVT_001` - `AVT_050`)
   - แสดงสถานะของเครื่องจักรด้วยสัญลักษณ์สีระบุความพร้อมใช้งานชัดเจน (พร้อมใช้งาน / ไม่พร้อมใช้งาน)

4. **คิวงานและการติดตามสถานะ AGV (Task Queue & AGV Status Tracking)**
   - ติดตามความคืบหน้าของงานแบบเรียลไทม์ผ่านขั้นตอนสถานะอย่างละเอียด: `Submitted` → `Queued` → `Starting` → `Moving to Source` → `Arrived at Source` → `Picking Up FPC` → `Waiting for Cover Head Installation` → `Moving to Destination` → `Arrived at Destination` → `Placing FPC` → `Waiting for Cover Head Removal` → `Completed`
   - จำลองการโต้ตอบหน้างานผลิตที่ต้องการการยืนยันจริงจากผู้ควบคุม:
     - **ยืนยันติดตั้งฝาครอบ (Confirm Cover Head Installed)**: ผู้ปฏิบัติงานทำการตรวจสอบฝาครอบเครื่องจักร และยืนยันความปลอดภัยโดยการกดปุ่มจริงที่ตัวรถ AGV
     - **ยืนยันถอดฝาครอบ (Confirm Cover Head Removed)**: ผู้ปฏิบัติงานตรวจสอบฝาครอบเครื่องจักร และยืนยันความปลอดภัยโดยการกดปุ่มจริงที่ตัวรถ AGV

5. **ระบบแผงควบคุมแอดมินและประวัติบันทึก (Admin Panel & Audit Logs)**
   - ดูประวัติเหตุการณ์ต่าง ๆ และล้างบันทึกการทำงานของระบบ (Audit Logs)
   - จัดการข้อมูล เพิ่ม ลบ หรือแก้ไขบัญชีผู้ใช้งาน (เฉพาะบทบาท Admin)
   - ปรับปรุงข้อมูลตำแหน่งของ FPC ได้ทันทีในตารางฐานข้อมูลจำลอง (เฉพาะบทบาท Admin และ Store)

## 📁 โครงสร้างโฟลเดอร์ของโครงการ (Directory Structure)

```text
src/
├── features/               # โมดูลฟังก์ชันหลักแยกตามขอบเขตงาน (Domain-specific feature modules)
│   ├── admin/              # แผงควบคุมผู้ดูแลระบบและบันทึกประวัติการทำงาน
│   ├── auth/               # ระบบล็อกอินของพนักงานและจัดการล็อกเอาต์
│   ├── queue/              # ระบบติดตามคิวงานและรายละเอียดใบงาน
│   └── workflow/           # หน้าเลือกโหมด, ฟังก์ชันเลือกเครื่องจักร และแบบฟอร์ม LOAD/UNLOAD/Swap/เปลี่ยน FPC
├── shared/                 # โค้ดส่วนกลาง คอมโพเนนต์ และ Asset ที่ใช้ร่วมกันทั้งหมด
│   ├── components/         # คอมโพเนนต์ UI ส่วนกลาง (ปัจจุบันเป็นโฟลเดอร์ว่าง)
│   ├── types/              # ไฟล์ประกาศอินเทอร์เฟซและไทป์สำหรับ TypeScript
│   └── utils/              # พจนานุกรมคำแปลภาษา และเลเยอร์ระบบจำลอง API (Mock API)
├── styles/                 # โครงสร้างการตกแต่งสไตล์ของโครงการ
│   ├── globals.css         # CSS ปรับแต่งการตั้งค่าเบื้องต้นและ Layout
│   ├── tailwind.css        # จุดนำเข้าไฟล์สไตล์ Tailwind CSS
│   └── theme.css           # การประกาศ CSS Tokens สำหรับ `@theme` คัสตอม
├── assets/                 # เก็บ Asset ต่าง ๆ (รูปภาพ, สัญลักษณ์ SVG)
├── App.tsx                 # คอมโพเนนต์หลักที่ควบคุมเส้นทางการเปลี่ยนหน้า (Routing)
├── App.css                 # สไตล์เฉพาะสำหรับคอมโพเนนต์หลัก
├── main.tsx                # จุดเริ่มต้นในการเรนเดอร์แอปพลิเคชัน (Entry point)
└── index.css               # สไตล์ตั้งต้นรวมของระบบ Tailwind และธีม
```

## 🛠️ ขั้นตอนเริ่มต้นใช้งาน (Getting Started)

### ข้อมูลรหัสผ่านสำหรับเข้าทดสอบระบบ (Default Test Credentials)
| บทบาท (Role) | รหัสพนักงาน (Employee ID) | รหัสผ่าน (Password) |
|---|---|---|
| Admin | `admin` | `admin` |
| Store | `store` | `store` |
| Operator | `op` | `op` |

*(หมายเหตุ: รหัสพนักงานและรหัสผ่านสำหรับบทบาท Operator ได้ถูกเปลี่ยนแปลงเพื่อทดสอบเป็น `op` ในไฟล์กำหนดสภาพแวดล้อม `.env` ในเวอร์ชันปัจจุบัน)*

### 1. ติดตั้งแพ็กเกจไลบรารีที่จำเป็น (Install Dependencies)
```bash
npm install
```

### 2. รันเซิร์ฟเวอร์สำหรับทดสอบพัฒนา (Run Local Development Server)
รันเว็บเซิร์ฟเวอร์บนพอร์ต **3000** (ถูกระบุไว้ใน [vite.config.ts](file:///c:/Users/kasid/OneDrive/Desktop/For_Gemini/Operator_UI/vite.config.ts) เพื่อป้องกันการชนกันกับพอร์ตโปรเจกต์อื่น ๆ)
```bash
npm run dev
```

### 3. ตรวจสอบไวยากรณ์และคุณภาพโค้ด (Lint Code)
```bash
npm run lint
```

### 4. คอมไพล์ซอร์สโค้ดเพื่อส่งมอบใช้งานจริง (Build for Production)
จัดระเบียบและคอมไพล์โค้ดให้ออกมาเป็นไฟล์รูปแบบ Static ขนาดเล็กพร้อมรันบนเซิร์ฟเวอร์ผลิตจริง ภายใต้โฟลเดอร์ `/dist`
```bash
npm run build
```
